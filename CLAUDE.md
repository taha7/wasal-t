# wasal-t — Claude Instructions

## Dev Log (mandatory)

After completing **every task, feature, or bug fix**, append an entry to `DEVLOG.md` at the repo root **before** reporting the task as done.

Each entry must include:
- Phase/task name and date
- Technologies or tools chosen (and why, if a decision was made)
- Every **file created or modified** (full relative path)
- Every **directory created**
- Every **package installed** with its resolved version
- **Functions or features implemented**, and in which files
- **Config values** set (env vars, ports, credentials, flags)
- **Commands run** and their outcome (containers started, migrations applied, etc.)

This log is the permanent audit trail of the project. It must be granular enough that anyone reading it can reconstruct exactly what was built, in what order, and why.
